1.讲一下你理解的Redis，为什么Redis很快

redis是c语言编写的非关系型数据库，它的数据是存放在内存中的

  • 因为它的结构简单
  • 基于内存读写
  • 采用复用型i/o模型，减少网络i/o的时间消耗，避免大量的无用操作
  • 采用单线程，避免了线程切换和上下文切换产生的消耗

2.你常用的Redis的数据存储结构有哪些，他们的使用场景分别是什么

常用类型有String，Zset，Hash

String用来存放用户登陆时的手机验证码，token之类的

Zset用来做排行

Hash用来做一些数据的缓存

3.每种存储结构说 4 个命令吧

String：

  • set key value 添加一行数据
  • get key 获取数据
  • mset key value key value。。。 批量添加数据据
  • mget key key。。。 批量获取数据

list：

  • lpush key value value。。。 从左边插入数据
  • rpush key value value。。。 从右边插入数据
  • lrange key start stop 查询区间的数据
  • lpop key 移出最左边的数据

set：

  • sadd key value value。。。 添加数据

  • smembers key 获取key中的所有数据

  • srem key member 删除key中的member元素

  • scard key 查询key’中元素个数

zset：

  • zadd key score value score value。。。 添加元素
  • zcard key 查询key中的元素个数
  • zcount key min max 返回集合中score在min和max范围中的元素
  • zrange key start stop 返回集合中的索引在start和stop间的元素

hash：

  • hset key feild value 添加数据

  • hget key feild 获取key中feild字段的数据

  • hmset key feild value feild value 批量添加数据

  • hmget key feild feild。。。 批量获取数据

4.你们项目是怎么用Redis的？

我们项目是在用户登录时获取验证码时将验证码存放到redis的，同时将用户的token也存放到redis，因为采用分布式部署，session就失效了，所以我们用redis模拟了一个session的功能，还有就是用redis做一些数据的缓存

5.怎么防止Redis宕机数据丢失问题

可以使用redis的持久化，将redis中的数据持久化到硬盘中，这样即使宕机也能保证数据不会丢失

6.Redis持久化是什么？有几种方式

redis的持久化就是将redis中的数据存放到硬盘中，有两种方式

  • AOF：存储命令的形式，就是当对redis进行操作时，会将操作的同步下来，也可以设置为每秒同步
  • RDB：数据快照，redis每隔一段时间就会将当前的数据进行一次保存

7.Redis有了AOF持久化为什么还要RDB？

因为使用AOF恢复数据如果涉及到数据量很大，那么AOF恢复数据的速度就会很慢，而使用RDB进行大数据恢复的时候就会相对AOF更快，但是AOF记录命令采用追加的方法，比起RDB更不容易涉及到数据丢失的情况，所以我们通常是将二者结合起来使用

8.Redis内存不够了怎么办？

有两种方法

  • 1.增加物理内存
  • 2.使用淘汰策略淘汰掉一些老旧的不经常使用的数据

9.淘汰策略有哪些？你们用的哪种

  • 从已设置过期时间的数据中淘汰快要过期的数据
  • 从已设置过期时间的数据中随机淘汰
  • 从已设置过期时间的数据中淘汰使用少的数据
  • 从数据中淘汰最少使用的数据
  • 从数据中随即淘汰
  • 不使用淘汰策略

10.Redis事务和Mysql事务的区别

  • redis的事务是不具备原子性的，而mysql的事务是具备原子性的，也就是说在mysql中一个service中有多条操作数据库的代码，如果其中一行报错，那么所有已经执行的代码都会回滚，但是redis中的不会，已经执行的代码不会回滚

  • mysql的事务是基于日志，记录修改数据前后的状态来实现的，而redis的事务则是基于队列实现的
