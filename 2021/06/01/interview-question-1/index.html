String和StringBuilder和StringBuffer区别

String的底层是字符数组,所以String是不可变字符串,每次改变就是创建新的字符串

StringBuilder和StringBuffer是可变字符串

但是StringBuilder是线程不安全的,StringBuffer是线程安全的,方法中使用synchronized来保证线程同步,所以StringBuiler的效率高于StringBuffer

ArrayList和LinkedList区别

ArrayList:是基于数组的集合,当执行插入和删除元素时会涉及到移位操作,所以相对来说,添加删除性能低

LinkedList:基于链表的集合,因为没有索引,所以查询元素时会遍历元素,所以查询修改性能低

多表查询，使用JOIN查询和额外SQL方式查询有什么区别，和优缺点

  • JOIN方式:在查询的数据的结构确定的情况以及结构层级较少时推荐使用这种方式,使用这种方式查询数据只需要向数据库发送一条sql,当查询数据的层级较多或层数不确定时join的次数太多,或者根本就不确定join的次数

  • 额外发送sql方式:查询的数据层级多或者不确定时使用这种方式,这种方式会将第一条sql查询到的数据作为参数传入到额外的sql作为参数进行查询,这样就能避免层级不确定的情况

Mybateis一级缓存和二级缓存的区别

缓存是指将数据存储起来,当用户在特定条件下获取数据时,不用去数据库查询,而是从缓存中调用数据

  • Mybatis一级缓存:一级缓存的作用域是SqlSession,当使用同一个sqlSession发送同一条sql时命中,在sqlSession销毁时清除缓存,sqlSession调用close()方法时销毁

  • Mybatis二级缓存:二级缓存的作用域是SqlSessionFactory,当使用同一个SqlSessionFactory发送相同的sql时命中,应在数据库执行添加,修改,删除后清除缓存

说一下会对AOP的理解，AOP的使用场景，是什么，AOP的实现原理是什么

  • AOP:面向切面思想,面向切面是对面向对象补充,而不是要取代面向对象,面向切面的核心思想是一段代码要作用在多个方法中,使用传统的面向对象思想就会使代码很臃肿,我们将方法可以看成一个点,而多个方法就可以看成一个面,我们可以在多个方法中作用相同的代码,比如方法执行前打印日志,管理事务等等.AOP的实现原理是基于java的动态代理进行实现

说一下切面相关的注解，分别是什么意思

  • @AfterThrowing:异常通知,使用在方法上,在原生方法抛出异常时执行的方法

springbean创建bean有两种形式,一种是单列模式,一种是多例模式

当被指定为singleton时为单例模式,spring会创建一个对象放进Map中,当需要使用这个bean的时候就将map中的对象返回

介绍一下Spring框架的组成

Data Access/Integration（数据访问／集成）

数据访问/集成层包括 JDBC、ORM、OXM、JMS 和 Transactions 模块，具体介绍如下。

  • JDBC 模块：提供了一个 JDBC 的抽象层，大幅度减少了在开发过程中对数据库操作的编码。
  • ORM 模块：对流行的对象关系映射 API，包括 JPA、JDO、Hibernate 和 iBatis 提供了的集成层。
  • OXM 模块：提供了一个支持对象/XML 映射的抽象层实现，如 JAXB、Castor、XMLBeans、JiBX 和 XStream。
  • JMS 模块：指 Java 消息服务，包含的功能为生产和消费的信息。
  • Transactions 事务模块：支持编程和声明式事务管理实现特殊接口类，并为所有的 POJO。

Web 模块

Spring 的 Web 层包括 Web、Servlet、Struts 和 Portlet 组件，具体介绍如下。

  • Web 模块：提供了基本的 Web 开发集成特性，例如多文件上传功能、使用的 Servlet 监听器的 IoC 容器初始化以及 Web 应用上下文。
  • Servlet模块：包括 Spring 模型—视图—控制器（MVC）实现 Web 应用程序。
  • Struts 模块：包含支持类内的 Spring 应用程序，集成了经典的 Struts Web 层。
  • Portlet 模块：提供了在 Portlet 环境中使用 MV C实现，类似 Web-Servlet 模块的功能。

Core Container（核心容器）

Spring 的核心容器是其他模块建立的基础，由 Beans 模块、Core 核心模块、Context 上下文模块和 Expression Language 表达式语言模块组成，具体介绍如下。

  • Beans 模块：提供了 BeanFactory，是工厂模式的经典实现，Spring 将管理对象称为 Bean。
  • Core 核心模块：提供了 Spring 框架的基本组成部分，包括 IoC 和 DI 功能。
  • Context 上下文模块：建立在核心和 Beans 模块的基础之上，它是访问定义和配置任何对象的媒介。ApplicationContext 接口是上下文模块的焦点。
  • Expression Language 模块：是运行时查询和操作对象图的强大的表达式语言。

其他模块

Spring的其他模块还有 AOP、Aspects、Instrumentation 以及 Test 模块，具体介绍如下。

  • AOP 模块：提供了面向切面编程实现，允许定义方法拦截器和切入点，将代码按照功能进行分离，以降低耦合性。
  • Aspects 模块：提供与 AspectJ 的集成，是一个功能强大且成熟的面向切面编程（AOP）框架。
  • Instrumentation 模块：提供了类工具的支持和类加载器的实现，可以在特定的应用服务器中使用。
  • Test 模块：支持 Spring 组件，使用 JUnit 或 TestNG 框架的测试。

你们项目是怎么用Redis的？

我们项目是在用户登录时获取验证码时将验证码存放到redis的，同时将用户的token也存放到redis，因为采用分布式部署，session就失效了，所以我们用redis模拟了一个session的功能，还有就是用redis做一些数据的缓存

怎么防止Redis宕机数据丢失问题

可以使用redis的持久化，将redis中的数据持久化到硬盘中，这样即使宕机也能保证数据不会丢失

Redis持久化是什么？有几种方式

redis的持久化就是将redis中的数据存放到硬盘中，有两种方式

  • AOF：存储命令的形式，就是当对redis进行操作时，会将操作的同步下来，也可以设置为每秒同步
  • RDB：数据快照，redis每隔一段时间就会将当前的数据进行一次保存

.Redis有了AOF持久化为什么还要RDB？

因为使用AOF恢复数据如果涉及到数据量很大，那么AOF恢复数据的速度就会很慢，而使用RDB进行大数据恢复的时候就会相对AOF更快，但是AOF记录命令采用追加的方法，比起RDB更不容易涉及到数据丢失的情况，所以我们通常是将二者结合起来使用

Redis内存不够了怎么办？

有两种方法

  • 1.增加物理内存
  • 2.使用淘汰策略淘汰掉一些老旧的不经常使用的数据

淘汰策略有哪些？你们用的哪种

  • 从已设置过期时间的数据中淘汰快要过期的数据
  • 从已设置过期时间的数据中随机淘汰
  • 从已设置过期时间的数据中淘汰使用少的数据
  • 从数据中淘汰最少使用的数据
  • 从数据中随即淘汰
  • 不使用淘汰策略

Redis事务和Mysql事务的区别

  • redis的事务是不具备原子性的，而mysql的事务是具备原子性的，也就是说在mysql中一个service中有多条操作数据库的代码，如果其中一行报错，那么所有已经执行的代码都会回滚，但是redis中的不会，已经执行的代码不会回滚

  • mysql的事务是基于日志，记录修改数据前后的状态来实现的，而redis的事务则是基于队列实现的

  • 微信将授权码返回到后台，后台将授权码重定向到前端页面
  • 前端页面再将授权码发送到后台，后台通过授权码获取用户的token，通过token获取到用户的信息，通过openId去数据库查询用户是否存在，如果存在就获取平台用户信息，将用户信息保存到redis中，使用uuid作为key，将用户信息返回给前端，前端将信息保存到LocalStroge，每次发请求都将信息存放到请求头中。
  • 如果数据库中没有用户的openId，就将用户信息保存到微信表中，让用户跳转到绑定页面
  • 用户输入手机号进行绑定，如果手机号已存在，就将用户id添加到微信表中的用户信息中，如果不存在就创建一个用户信息，将id保存到微信表中的用户信息中

为什么要使用连接池，数据库连接池的原理

如果不使用连接池那么每次操作数据库时都会创建连接，使用完之后都需要关闭连接，性能比较低，而使用数据库连接池则不需要每次都创建连接和关闭连接，而是每次使用都会从连接池中获取连接，使用完后再将连接放回到连接池，避免了每次创建连接和关闭连接带来的性能损耗。

数据库连接池的原理是用数据库连接池来对连接进行管理，需要连接对象时就从连接池中获取，使用完之后就将连接池放回到连接池中

创建线程的3种方式, 三种线程的区别？start() 和 run()方法区别

方式一：创建类集成Thread，覆写run方法，创建类的实例，调用start()方法启动线程

方式二：创建类实现Runnable，覆写run方法，创建类的实例，将对象作为参数创建Thread对象，调用Thread对象的start()方法

方式三：创建类实现Callable，复习call方法，创建类的实例，将对象作为参数创建FutureTask对象，将FutureTask对象作为参数创建Thread对象，调用Thread对象的run()方法启动线程，调用FutureTask对象的get方法获取线程返回值

start()是创建一个线程来执行run()方法，run()则是直接调用run()方法，不会开启新的线程

Synchronized和Lock的区别

Synchronized：执行锁是隐式的，使用简单，不需要我们去关心锁的开启与释放，层次是JVM上的，不可获取锁状态。

Lock：面向对象，通过调用方法来获取锁和释放锁，层次是类上，可以获取锁状态

线程的生命周期,sleep 和 wait区别

线程被创建->线程可运行状态(Runnable)->线程运行状态->线程锁状态->线程死亡

sleep：当前线程休眠，会自动唤醒，不会释放锁，需要抛异常

wait：不会自动唤醒，需要其他线程调用notify或则notifyAll才能唤醒，会释放锁

synchronized 锁的原理，JDK1.6之后做了什么样的优化?

sysnchronized锁的原理JVM内置锁，通过内部对象Monitor实现，依赖于底层操作系统Mutex lock实现

jdk1.6以后使用了偏向锁，轻量级锁，重量级锁

偏向锁：线程并发获取一把锁的情况是很少的，一般为一个线程获取一把锁执行代码，这个线程可以重复获取当前锁，多次获取锁时并不会重复加锁，消除了锁重入的开销，结果至始至终使用锁的线程只有一个，所以偏向锁基本没有开销，性能很高

轻量级锁：当锁出现线程竞争时，偏向锁会升级成轻量级锁，没有获取到锁的线程自旋

重量级锁：当出现锁竞争时，没有获取到锁的线程自选超过了默认值（10）后，轻量级锁会升级为重量级锁，重量级锁是jdk5之前使用的锁，非常影响性能

什么是乐观锁，什么是悲观锁，什么是重入锁，什么是自旋锁，怎么是阻塞。

乐观锁不会在代码中加锁，认为操作数据时不会有其他线程来修改，但是在提交修改的时候会判断数据是否被修改了

悲观锁认为只要操作数据就会有其他数据来操作，直接在代码开始加锁，代码结束释放锁

重入锁是指当方法获取到锁之后还可以再次获取到当前锁

自旋锁是指当方法获取锁后其他方法来获取锁被阻拦了，然后它可以每隔一段时间来尝试获取锁

阻塞是指当锁被其他方法获取后，后来的方法就会被阻塞，它会一直等待锁被释放，直到它获取到锁

说一下散列存储(Hash存储) ， 什么是Hash冲突 ， 有什么解决方案

hash存储是通过hash算法将要存储的数据经过计算来模于数组长度得到存储的位置，将数据存放到数组中

hash冲突指的是不同的值通过hash算法计算出来的值相同，这时存储的位置就会发生冲突

解决方法：

1.产生hash冲突的值使用链表挂在数组上

2.对冲突的值再次使用hash计算得到新的值来存储

3.准备多套hash算法，产生冲突时就换其他算法进行计算

4.将冲突的数据放到其他数组

举例说说时间复杂度，比如：数组，链表，循环，嵌套循环 ， 2个非嵌套循环 ， 在什么情况下时间复杂度是什么？

时间复杂度指的是对数据进行操作所需的时间难度

使用数组获取指定值的时间难度是T(1)，因为可以使用索引来直接过去

使用链表来获取值的时间难度是T(n)，通过链表获取值需要遍历链表，最极端的情况需要遍历所有的数据才能获得需要的数据

使用循环来获取值的时间难度是T(n)，最极端的情况需要循环完才能获得数据

嵌套循环获取值的时间难度是T(n^2)，最极端的情况需要遍历完两个循环才能获得数据

两个非嵌套循环获取值的是按难度是T(2n)，最极端的情况是两个循环遍历完

说一下树的分类，以及你对它们的理解（二叉查找树的优缺点，平衡树的优缺点，红黑树的优缺点，B-树的优缺点 ，B+树的优缺点）平衡树 ，红黑树

二叉树：只有一个根节点，每个节点可以有最多两个子节点，每个子节点只有一个兄弟节点，每个节点左边的子节点比这个节点的值小，右边的子节点比这个节点的值大

  • 优点：结构相较于其他树比较简单，树状结构查询数据速度快
  • 缺点：极端情况下可能会变成链表，性能变低

平衡树：在二叉树的基础上能够自动平衡树结构，不会出现子节点的节点数差值超过1

  • 优点：比起二叉树不会出现链表结构，树状结构绝对平衡，查询性能极高
  • 缺点：每次添加修改删除元素后，平衡树需要自动平衡节点，保持绝对平衡，所以性能低，不适用于频繁增删改操作元素的场景

红黑树：保证每个叶子节点到根节点中的黑节点数相同，不会连续出现两个红节点的情况

  • 优点：相对于平衡树提升了增删改的性能
  • 缺点：相对于平衡树降低了查询性能

B-树：允许每个节点存放的元素大于两个，存放的元素越多数的高度越低，拥有所有二叉树的特性

  • 优点：减少了树状结构的高度，能够存储更多的数据
  • 缺点：不支持范围查找和排序

B+树：在B-树的基础上对叶子节点的数据进行了维护，将所有叶子节点的数据存放到链表中

有是二叉树为什么要出现多叉

二叉树不适合用来存储大量数据，存储大量数据会让树结构变得很高，性能变低

多叉树就解决了这个问题，可以用来存储大量数据

b-tree和b+tree的区别？

B-树是在存储的索引和值，B+树是在节点存储索引，在叶子节点存储值，且将值封装到有序链表中，方便范围查询和排序

RabbitMQ的使用场景

主要运用在高并发中，用来做一个异步的方法调用。

服务解耦，服务之间不通过直接调用，而是通过RabbitMQ生产者发消息，使用消费者来消费消息

RabbitMQ如何防止消息丢失

让消息持久化，避免消息因为RabbitMQ宕机或者网络波动而导致丢失

设置签收模式为手动签收，避免消息消费失败却被签收了

RabbitMQ的交换机有哪几种

Fanout：广播，将消息交给所有绑定到交换机的队列

Direct：定向，把消息交给符合指定routing key的队列

Topic：通配符，把消息交给符合routing pattern的队列

消息是如何从发送者到达消费者的（RabbitMQ工作流程）

分为消息发送和消息接收两个步骤：

  • 消息发送：生产者和Broker建立TCP连接，创建信道。通过信道将消息发送给Broker，由Exchange将消息进行转发到指定的队列

  • 消息接收：消费者和Broker建立TCP连接 ，创建信道 ，然后监听指定的队列，当有消息到达队列时，Broker默认将消息推送给消费者，消费者就能接收到消息

如何防止消息重复消费？

可以在发送消息的时候将一个标记存放到数据库，在消费成功后去数据库删除这个标记，如果在消费之前数据库中没有这个标记就说明消息已经被消费了，直接丢弃就行

Lucene创建索引原理

Lucene创建倒排索引的原理是：

  • 首先将数据进行一个分词处理

什么是集群

集群是指将项目或者服务用多台服务器来部署，每个服务器上的项目和服务都是完整的

什么是负载均衡

做了集群后，项目或者服务就会出现多个不同的ip，用户去访问的时候就不知道该访问哪一个，我们在服务之间的调用也不能确定该调用哪一个，负载均衡就是用来解决这个问题的，它可以通过算法来让我们发起请求时访问到项目或服务

什么是分布式

分布式是指将项目拆分成多个服务，将不同的服务部署到不同的服务器上，每个服务独立运行

集群和分布式的区别，分别解决什么问题

集群是将一个项目或者一个服务部署到多个服务器上，而分布式是将一个项目拆分成多个服务部署到多个服务器上

集群中有一个服务器宕机了也不会影响业务流程，项目依然能够使用，但是分布中一个服务器宕机了，那么这个服务就不能使用了，但是其他服务任然能使用

说一下你理解的微服务

我理解的微服务就是将一个项目拆分成多个服务，降低了项目的耦合度，让项目变得不那么臃肿。并且后期添加功能也会更方便。缺点就是分布式事务复杂，服务之前通行的性能损耗，技术成本高，部署麻烦

讲一种微服务解决方案

使用SpringBoot NetFlix作为基础微服务架构

Eureka来作为服务发现与注册

Zull来作为服务网关

使用Feign来做为服务通信与负载均衡

使用Hystrix来处理服务熔断，解决单节点故障

使用Config分布式配置中来统一管理配置微服务配置

使用Bus消息总线来给各个微服务广播消息，可以实现微服务配置的自动刷新

使用Sleuth链路追踪，来实时监控各个微服务建的调用关系，快速定位故障节点

Spring Cloud的优缺点？

微服务相对单体应用来说

优点：

  • 服务之间无耦合，代码简单方便开发维护，服务之间升级维护互不影响
  • 轻量级HTTP通信机制，不同的服务可以采用不同的编程语言
  • 有极强的扩展能力,业务量大的服务可以再次拆分服务，或者也可以集群部署
  • 支持时下流行的敏捷开发并做了优化

缺点：

  • 分布式事务繁琐
  • 部署麻烦，开发人员的学习成本高
  • 技术成本高，开发人员需要花更多的时间学习相关技术
  • 微服务间的通信存在对性能的损耗问题
