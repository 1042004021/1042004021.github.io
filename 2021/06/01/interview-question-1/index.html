1.String和StringBuilder和StringBuffer区别

String的底层是字符数组,所以String是不可变字符串,每次改变就是创建新的字符串
StringBuilder和StringBuffer是可变字符串
但是StringBuilder是线程不安全的,StringBuffer是线程安全的,方法中使用synchronized来保证线程同步,所以StringBuiler的效率高于StringBuffer

2.ArrayList和LinkedList区别

ArrayList:是基于数组的集合,当执行插入和删除元素时会涉及到移位操作,所以相对来说,添加删除性能低
LinkedList:基于链表的集合,因为没有索引,所以查询元素时会遍历元素,所以查询修改性能低

3.创建线程是几种方式

创建线程有三种方式:
方式一:通过继承Thread
方式二:通过实现Runnable
方式三:通过实现Callable

4.直接调用线程的start方法和run方法有什么区别

调用start方法是创建一个新线程,让它拥有获取cpu资源的能力

7.ArrayList和vector区别

ArrayList和vector都是基于数组的集合,而vector是线程安全的,底层方法中有synchronize修饰,ArrayList则是不保证线程安全
在扩容方面二者也有区别,ArrayList添加数据当容量不足时自动扩容到当前容量的1.5倍,而vector则是扩容到当前容量的2倍

继承:继承是当前类中的属性以及方法不能满足使用的需求时,我们便通过继承来获取这个类的所有属性,并在其基础上对其进行扩展
多态:多态是指程序在编译和运行时表现出不同的形态,意义在于屏蔽子类差异

15.多表查询，使用JOIN查询和额外SQL方式查询有什么区别，和优缺点

  • JOIN方式:在查询的数据的结构确定的情况以及结构层级较少时推荐使用这种方式,使用这种方式查询数据只需要向数据库发送一条sql,当查询数据的层级较多或层数不确定时join的次数太多,或者根本就不确定join的次数
  • 额外发送sql方式:查询的数据层级多或者不确定时使用这种方式,这种方式会将第一条sql查询到的数据作为参数传入到额外的sql作为参数进行查询,这样就能避免层级不确定的情况

18.Mybateis一级缓存和二级缓存的区别

缓存是指将数据存储起来,当用户在特定条件下获取数据时,不用去数据库查询,而是从缓存中调用数据
  • Mybatis一级缓存:一级缓存的作用域是SqlSession,当使用同一个sqlSession发送同一条sql时命中,在sqlSession销毁时清除缓存,sqlSession调用close()方法时销毁
  • Mybatis二级缓存:二级缓存的作用域是SqlSessionFactory,当使用同一个SqlSessionFactory发送相同的sql时命中,应在数据库执行添加,修改,删除后清除缓存

21.说一下会对AOP的理解，AOP的使用场景，是什么，AOP的实现原理是什么

  • AOP:面向切面思想,面向切面是对面向对象补充,而不是要取代面向对象,面向切面的核心思想是一段代码要作用在多个方法中,使用传统的面向对象思想就会使代码很臃肿,我们将方法可以看成一个点,而多个方法就可以看成一个面,我们可以在多个方法中作用相同的代码,比如方法执行前打印日志,管理事务等等.AOP的实现原理是基于java的动态代理进行实现

22.说一下切面相关的注解，分别是什么意思

  • @aspect:定义切切面定义在类上,表明这个类中的方法作用在切面上
  • @pointCut:定义切点,参数为需要使用切面的包,包中类的方法都会应用切面
  • @before:前置通知,使用在方法上,在原生方法执行前需要执行的方法
  • @after:后置通知,使用在方法上,在原生方法执行完后执行的方法
  • @afterReturning:最终通知 ,使用在方法上,在原生方法正常结束后执行的方法
  • @AfterThrowing:异常通知,使用在方法上,在原生方法抛出异常时执行的方法

25.Spring的Bean被指定为prototype以及singleton有什么区别

当被指定为singleton时为单例模式,spring会创建一个对象放进Map中,当需要使用这个bean的时候就将map中的对象返回

28.介绍一下Spring框架的组成

29. Data Access/Integration（数据访问／集成）

数据访问/集成层包括 JDBC、ORM、OXM、JMS 和 Transactions 模块，具体介绍如下。

  • JDBC 模块：提供了一个 JDBC 的抽象层，大幅度减少了在开发过程中对数据库操作的编码。
  • ORM 模块：对流行的对象关系映射 API，包括 JPA、JDO、Hibernate 和 iBatis 提供了的集成层。
  • OXM 模块：提供了一个支持对象/XML 映射的抽象层实现，如 JAXB、Castor、XMLBeans、JiBX 和 XStream。
  • JMS 模块：指 Java 消息服务，包含的功能为生产和消费的信息。
  • Transactions 事务模块：支持编程和声明式事务管理实现特殊接口类，并为所有的 POJO。

30. Web 模块

Spring 的 Web 层包括 Web、Servlet、Struts 和 Portlet 组件，具体介绍如下。

  • Web 模块：提供了基本的 Web 开发集成特性，例如多文件上传功能、使用的 Servlet 监听器的 IoC 容器初始化以及 Web 应用上下文。
  • Servlet模块：包括 Spring 模型—视图—控制器（MVC）实现 Web 应用程序。
  • Struts 模块：包含支持类内的 Spring 应用程序，集成了经典的 Struts Web 层。
  • Portlet 模块：提供了在 Portlet 环境中使用 MV C实现，类似 Web-Servlet 模块的功能。

31. Core Container（核心容器）

Spring 的核心容器是其他模块建立的基础，由 Beans 模块、Core 核心模块、Context 上下文模块和 Expression Language 表达式语言模块组成，具体介绍如下。

  • Beans 模块：提供了 BeanFactory，是工厂模式的经典实现，Spring 将管理对象称为 Bean。
  • Core 核心模块：提供了 Spring 框架的基本组成部分，包括 IoC 和 DI 功能。
  • Context 上下文模块：建立在核心和 Beans 模块的基础之上，它是访问定义和配置任何对象的媒介。ApplicationContext 接口是上下文模块的焦点。
  • Expression Language 模块：是运行时查询和操作对象图的强大的表达式语言。

32. 其他模块

Spring的其他模块还有 AOP、Aspects、Instrumentation 以及 Test 模块，具体介绍如下。

  • AOP 模块：提供了面向切面编程实现，允许定义方法拦截器和切入点，将代码按照功能进行分离，以降低耦合性。
  • Aspects 模块：提供与 AspectJ 的集成，是一个功能强大且成熟的面向切面编程（AOP）框架。
  • Instrumentation 模块：提供了类工具的支持和类加载器的实现，可以在特定的应用服务器中使用。
  • Test 模块：支持 Spring 组件，使用 JUnit 或 TestNG 框架的测试。

文章来源:Spring体系结构详解 (biancheng.net)

58.你们项目是怎么用Redis的？

我们项目是在用户登录时获取验证码时将验证码存放到redis的，同时将用户的token也存放到redis，因为采用分布式部署，session就失效了，所以我们用redis模拟了一个session的功能，还有就是用redis做一些数据的缓存

59.怎么防止Redis宕机数据丢失问题

可以使用redis的持久化，将redis中的数据持久化到硬盘中，这样即使宕机也能保证数据不会丢失

60.Redis持久化是什么？有几种方式

redis的持久化就是将redis中的数据存放到硬盘中，有两种方式

  • AOF：存储命令的形式，就是当对redis进行操作时，会将操作的同步下来，也可以设置为每秒同步
  • RDB：数据快照，redis每隔一段时间就会将当前的数据进行一次保存

61.Redis有了AOF持久化为什么还要RDB？

因为使用AOF恢复数据如果涉及到数据量很大，那么AOF恢复数据的速度就会很慢，而使用RDB进行大数据恢复的时候就会相对AOF更快，但是AOF记录命令采用追加的方法，比起RDB更不容易涉及到数据丢失的情况，所以我们通常是将二者结合起来使用

62.Redis内存不够了怎么办？

有两种方法

  • 1.增加物理内存
  • 2.使用淘汰策略淘汰掉一些老旧的不经常使用的数据

63.淘汰策略有哪些？你们用的哪种

  • 从已设置过期时间的数据中淘汰快要过期的数据
  • 从已设置过期时间的数据中随机淘汰
  • 从已设置过期时间的数据中淘汰使用少的数据
  • 从数据中淘汰最少使用的数据
  • 从数据中随即淘汰
  • 不使用淘汰策略

64.Redis事务和Mysql事务的区别

  • redis的事务是不具备原子性的，而mysql的事务是具备原子性的，也就是说在mysql中一个service中有多条操作数据库的代码，如果其中一行报错，那么所有已经执行的代码都会回滚，但是redis中的不会，已经执行的代码不会回滚

  • mysql的事务是基于日志，记录修改数据前后的状态来实现的，而redis的事务则是基于队列实现的
